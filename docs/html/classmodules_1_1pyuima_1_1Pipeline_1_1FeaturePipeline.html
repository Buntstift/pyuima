Detailed Description

Basic text processing pipeline.

Todo:

add check function for pipeline

change 'fileids' to a flexible parameter

This class implements a rudimentary UIMA pipeline. The set-up comprises three steps: First, a text corpus is initialized. In addition, optional annotations can be specified through a pandas DataFrame. Second, functions that make use of the whole corpus are set. Third, functions that extract certain features are given. Finally, the pipeline can be run. The results can be obtained by several functions according to the preferred format.

Initialize corpus functions.

This function executes a list of functions (*funs) in the order they are provided in the list. Each function is given **args as additional arguments. The results are stored in a dictionary inside the class for later use.

Parameters:
*funsa list of functions in the following format: the first argument is a corpus. All other arguments are named arguments, that are provided in **args
**argslist of tuples containing the arguments for the functions listed in *funs

Initialize document functions.

This function initializes a list of functions to be executed for each document. Each function in *funs returns a feature from the current document.

Parameters:
*funsList of functions to be executed for each document. The functions follow a particular pattern.
**argsAdditional arguments as named arguments. These are given as arguments to each of the functions given in *funs. For example, they can be used to provide parameter values for the feature extractors.
def modules.pyuima.Pipeline.FeaturePipeline.runFeaturePipeline (   self)

Runs the pipeline for feature extraction.

All functions provide in the earlier steps of the set-up are executed. The named arguments specyfied are given as parameters for the function when executed. As a result, the features are computed as a list of tuples. Each entry of the list contains the features for a single document of the corpus. The results can be assessed by several functions.
